## 目的

- 品質水準の向上
- 観点だしの効率化
- 早い段階で品質面からの仕様を検討できるようになり、開発スピードに貢献

## QAフロー

1. PRDレビュー
   1. なぜやるか
   2. 何をやるか/やらないか
   3. PRDの各項目の重要度、優先度を確認
      1. PdM, Deがどこを必ず抑えたいか、必ず握っておくとスムーズに会話できます
2. FSD作成（仕様・スコープ）
   1. どうやって実装するか/実装できないか
      1. 表示
         1. 文言
         2. 順番
         3. 活性/非活性
      2. 動き
         1. ホバー/フォーカス時の表示
         2. クリック/ダブルクリック/連打/長押し
         3. ドラッグアンドドロップ
         4. エラー挙動
      3. エッジケース
      4. ...具体的な仕様を記載
   2. PRDの重要度に応じて、FSDの重要度を設定
3. テスト観点作成
   1. 目安は50~80項目
4. テスト設計
   1. テスト技法（境界値・同値）を活用し、観点をGherkin形式のテストケースに変換
      1. 目安は150~200項目
   2. FSDと1:1に対応するテストケースを追記
   3. リグレッションテストケースを追記
      1. 過去仕様から情報収集
         1. Notion MCP
         2. GitHub
      2. 過去テスト設計書から情報収集
         1. プレーンテキストではない場合が多いので、ここは効率良く情報収集する方法が必要かもしれません
5. テスト実施
